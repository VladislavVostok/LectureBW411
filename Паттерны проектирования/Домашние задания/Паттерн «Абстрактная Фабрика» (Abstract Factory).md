**Контекст:**

Вы разрабатываете модуль симуляции транспортной логистики. Система должна работать с двумя типами транспорта: **Грузовым** и **Пассажирским**. Каждый тип транспорта имеет свою собственную, несвязанную реализацию для двух разных сред передвижения: **Наземного** и **Морского**. Ваша задача — обеспечить создание согласованных семейств объектов (например, нельзя создать Грузовой корабль и Пассажирский поезд в одной логической группе), чтобы клиентский код мог работать с любыми вариациями, не зная их конкретных классов.

**Задача:**
	Реализовать паттерн «Абстрактная Фабрика» для создания семейств транспортных средств.

**Детали реализации:**

- **Определите Абстрактные Продукты**:
	Создайте абстрактные классы (интерфейсы) для каждого вида продукта в семействе.
	- ITransport: базовый интерфейс с методами deliver() и getType().
	- ITruck: наследуется от ITransport. Может иметь специфичный метод getCargoCapacity().
	- IShip: наследуется от ITransport. Может иметь специфичный метод getDisplacement().

- **Реализуйте Конкретные Продукты:**
	Создайте конкретные классы для всех вариаций продуктов.

	- Для наземного грузового транспорта: CargoTruck.
	- Для наземного пассажирского транспорта: PassengerVan.
	- Для морского грузового транспорта: CargoShip.
	- Для морского пассажирского транспорта: PassengerFerry.

- **Определите Абстрактную Фабрику:**
	Создайте интерфейс ITransportFactory, который объявляет набор методов для создания каждого абстрактного продукта (например, createTruck(), createShip()). Эти методы должны возвращать абстрактные типы продуктов.

- **Реализуйте Конкретные Фабрики:**
	Создайте классы-реализации абстрактной фабрики для каждой вариации семейства продуктов. Каждая фабрика соответствует определённой "логике" создания.

	- CargoTransportFactory: создает экземпляры CargoTruck и CargoShip.
	- PassengerTransportFactory: создает экземпляры PassengerVan и PassengerFerry.

  
- **Напишите Клиентский Код:**
	Код должен принимать объект абстрактной фабрики и использовать её для создания объектов транспорта. Он должен работать исключительно с интерфейсами ITruck и IShip, не завися от их конкретных реализаций.

**Схема:**
	Сделать UML диаграмму классов, для этого задания. Задание сдается в виде текстового документа формата .txt в корне вашего проекта на C++.
